# 8.2 系统架构评估

## 1. 本节核心思想（100字以内）

系统架构评估是在系统开发前，通过场景分析、质量属性分析和成本收益分析等方法，识别架构风险、敏感点和权衡点，验证架构是否满足性能、安全性、可修改性、可用性等质量目标，为架构决策提供依据。

---

## 2. 考试高频考点（尽量精简）

### 架构评估方法分类
- 问卷/检查表法
- 场景法（重点）
- 度量法

### 核心概念
- 敏感点：影响单个质量属性
- 权衡点：影响多个质量属性
- 场景：刺激 + 环境 + 响应
- 利益相关者（Stakeholder）

### SAAM
- 基于场景分析
- 重点评估：可修改性
- 流程：场景开发 → 架构描述 → 场景评估 → 场景交互 → 总体评估

### ATAM（重点）
- 基于 SAAM 扩展
- 关注：性能、安全性、可修改性、可用性
- 关注：敏感点、权衡点、风险点
- 工具：效用树（Utility Tree）
- 场景优先级：(重要性H/M/L，难易度H/M/L)

### CBAM
- 基于 ATAM
- 关注成本与收益
- 核心指标：ROI（投资回报率）
- 用于架构策略优先级排序

### AHP
- 层次分析法
- 两两比较
- 计算权重
- 多方案决策排序
