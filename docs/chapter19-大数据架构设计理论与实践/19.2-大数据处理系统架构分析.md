# 19.2 大数据处理系统架构分析

## 1. 本节核心思想（100字以内）

大数据处理的核心挑战是处理非结构化、半结构化、复杂且异构的数据，并通过“一次挖掘”获得粗糙知识，再结合经验和决策需求进行“二次挖掘”，形成智能知识。大数据系统应具备容错、低延迟、横向扩容、通用、易扩展、即席查询、少维护和可调试等特征。

## 2. 考试高频考点

- **考点1：大数据处理面临的三大挑战**  
  包括非结构化和半结构化数据处理、复杂性与不确定性的描述和建模，以及数据异构性与决策异构性之间的协调。

- **考点2：一次挖掘与二次挖掘**  
  一次挖掘从原始数据中发现潜在模式，形成“粗糙知识”；二次挖掘将粗糙知识与经验、常识、情境和用户偏好结合，形成“智能知识”。

- **考点3：大数据系统的八项架构特征**  
  包括鲁棒性和容错性、低延迟读取与更新、横向扩容、通用性、延展性、即席查询、最少维护和可调试性。

- **易混点：横向扩容与纵向扩容**  
  横向扩容（Scale Out）通过增加机器数量提高处理能力；纵向扩容（Scale Up）通过增强单台机器配置提高性能，大数据系统通常优先采用横向扩容。

- **易混点：可扩展性与延展性**  
  可扩展性侧重数据量或负载增加时通过增加资源维持性能；延展性侧重新需求出现时能够方便地增加功能或迁移系统。

- **易混点：数据异构性与决策异构性**  
  数据异构性是数据来源、格式、结构和语义不同；决策异构性是不同决策者的目标、经验和偏好不同，二次挖掘是连接二者的桥梁。

- **易考选择题：大数据系统容错的重点**  
  除机器宕机等硬件故障外，还要重视程序缺陷、错误数据和人为误操作，其中人为操作容错往往比机器容错更加重要。

- **易考选择题：即席查询的含义**  
  即席查询是用户根据临时需求自由设置查询条件，无须事先开发固定报表，强调数据分析的灵活性。

- **易考选择题：可调试性的含义**  
  系统产生的每一个结果都应能够追踪其数据来源、处理步骤和生成过程，而不仅仅是记录程序运行日志。
