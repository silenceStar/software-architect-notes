# 19.5 Lambda 架构与 Kappa 架构的对比和设计选择

## 1. 本节核心思想（100字以内）

Lambda 架构采用批处理层和速度层两套系统，历史数据处理能力强，但复杂度和维护成本高；Kappa 架构采用一套流处理系统，通过消息重放实现历史重算，结构简单、成本较低，但处理海量长期历史数据的能力相对较弱。

```text
Lambda：批处理层 + 速度层 → 结果合并
Kappa ：消息队列 → 流处理系统 → 统一结果
````

## 2. 考试高频考点

* **考点1：两种架构的核心区别**
  Lambda 需要分别维护批处理和流处理两套系统；Kappa 只维护一套流处理系统，历史重算通过重新消费消息完成。

* **考点2：架构选择依据**
  需要频繁处理海量历史数据、离线训练与实时预测无法统一时选 Lambda；强调实时处理、统一代码和低维护成本时选 Kappa。

* **易混点：Kappa 并非不能处理历史数据**
  Kappa 可以通过 Kafka 等消息队列重放历史数据，但在长期、海量历史数据回溯方面通常弱于 Lambda 的批处理层。

* **易考选择题：机器学习离线训练与实时预测应选哪种架构**
  离线批处理负责训练模型，实时流处理负责加载模型并预测，两者逻辑无法合并，通常应选择 Lambda 架构。

## 3. 记忆口诀

**两套强历史，一套低成本；历史多选 Lambda，流处理选 Kappa。**

解释：

* “两套强历史”：Lambda 维护批处理、流处理两套系统，历史数据处理能力强。
* “一套低成本”：Kappa 只维护一套流处理系统，开发和维护成本低。
* 海量历史数据频繁分析、全量回刷，优先选择 Lambda。
* 实时业务为主、算法频繁修改、希望批流统一，优先选择 Kappa。

## 4. 考试一句话总结（精简）

本节最可能考 Lambda 与 Kappa 在系统复杂度、维护成本、历史数据处理能力上的区别及架构选择。
