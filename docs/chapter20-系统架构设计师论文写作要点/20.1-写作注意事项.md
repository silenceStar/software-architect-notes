# 20.1 写作注意事项

## 1. 本节核心思想（100字以内）

系统架构设计师论文重点考查项目实践、问题分析与解决能力及表达能力。备考时应选择近几年有代表性的真实项目，提前整理项目背景、个人职责、架构设计、问题措施和实施效果，并通过“一个项目适配多个主题”的万能框架进行限时机考训练。

## 2. 考试高频考点

* **考点1：论文考查目的**
  重点考查考生是否具备系统架构设计实践经验、独立分析和解决问题的能力，以及准确清晰的表达能力。

* **考点2：以不变应万变**
  “不变”是提前准备的代表性项目及其背景、规模、职责和总体架构，“万变”是根据不同题目调整论文的技术重点。

* **易混点：论文不是理论知识堆砌**
  不能只写技术定义和优点，必须结合项目说明为什么采用、如何实施、遇到什么问题、如何解决以及最终效果。

* **易考选择题：论文备考方法**
  应重视平时积累、项目复盘和限时训练；经验丰富者重在总结升华，经验较少者可学习项目文档和案例，学生应通过范文和案例弥补实践不足。

* **实战万能模板：项目论文基本结构**
  论文可统一采用“项目背景与职责—技术分析—架构设计与实施—问题及解决—效果与总结”的结构，再根据题目替换技术主体。

### 万能论文框架

```text
一、项目背景与本人职责
1. 项目建设背景
2. 项目目标与规模
3. 系统主要功能
4. 本人在项目中的角色和职责

二、论文主题相关理论
1. 简要说明相关技术或架构思想
2. 分析该技术适用于本项目的原因
3. 说明选型时考虑的主要因素

三、架构设计与具体实施
1. 总体架构设计
2. 核心模块或关键技术设计
3. 数据、接口、部署或安全设计
4. 具体实施过程

四、项目中遇到的问题及解决措施
1. 出现了什么问题
2. 问题产生的原因
3. 采取了什么解决方案
4. 方案实施后的效果

五、项目效果与总结
1. 性能、可用性、质量或业务效果
2. 项目中存在的不足
3. 后续优化方向
4. 本人的经验与体会
```

### 万能叙述逻辑

```text
项目背景
    ↓
业务需求
    ↓
架构问题
    ↓
方案选型
    ↓
设计实施
    ↓
问题解决
    ↓
量化效果
    ↓
经验总结
```

### 单个技术点的万能写法

```text
由于……业务需求，原有系统存在……问题。
经过分析，我认为问题主要由……导致。
为此，我负责设计并采用……方案。
在实施过程中，重点完成了……工作。
针对实施中出现的……问题，又采取了……措施。
最终，系统的……指标由……提升到……，满足了项目要求。
```

> **机考修正：**教材中关于方格纸、用笔和手写速度的内容已不再适用。当前应重点训练键盘输入速度、快速审题、列提纲、段落组织和限定时间内完成整篇论文的能力。

## 3. 记忆口诀

**真项为基，职责写清；理论点题，实践为主；问题成线，效果收尾；一项多变，限时勤练。**

解释：

* **真项为基**：选择真实、熟悉且具有一定规模的项目；
* **职责写清**：明确本人承担的架构设计和关键技术工作；
* **理论点题**：理论只需简要说明，不能喧宾夺主；
* **实践为主**：重点写架构设计、实施过程和技术细节；
* **问题成线**：按照“问题—分析—方案—实施”展开；
* **效果收尾**：使用性能、数据质量或业务指标说明成果；
* **一项多变**：一个项目可以适配多种论文题目；
* **限时勤练**：通过机考模拟提升构思与输入速度。

## 4. 考试一句话总结（精简）

本节最可能考论文的考查目的、备考方法，以及如何使用“背景—职责—设计—问题—效果”万能框架做到以不变应万变。
