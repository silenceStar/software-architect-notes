# 9.5 软件可靠性测试

## 1. 本节核心思想（100字以内）

软件可靠性测试以真实运行环境为基础，通过构建运行剖面（Operational Profile），按用户实际使用概率设计测试用例，收集失效数据并进行可靠性评估。其目标不仅是发现缺陷，更是量化软件可靠性水平，为可靠性预测和验收提供依据。

---

## 2. 考试高频考点（尽量精简）

### （1）软件可靠性测试流程

```text
确定可靠性目标
→ 开发运行剖面
→ 设计测试用例
→ 实施测试
→ 分析测试结果
```

### （2）运行剖面（重点）

- 定义：用户真实使用行为的概率分布
- 建模方法：马尔可夫链（Markov）
- 作用：指导测试资源分配和测试用例设计

### （3）运行剖面分层

- 用户级分层（谁使用）
- 用法级分层（怎么使用）

### （4）使用概率来源（优先级）

```text
真实运行数据
＞ 用户调研
＞ 原型测试
＞ 专家经验
＞ 平均分配
```

### （5）关键功能测试

运行概率低但风险高的功能需单独测试：

- 边界情况
- 状态跃迁
- 关键功能
- 异常场景

### （6）可靠性测试用例组成

```text
标识
被测对象
测试环境
测试输入
操作步骤
预期输出
判定标准
特殊需求
```

### （7）可靠性测试重点场景

- 用户误操作
- 错误提示准确性
- 异常退出
- 数据可靠性
- 故障恢复能力

### （8）可靠性测试实施原则

- 必须受控测试
- 测试期间软件版本保持一致
- 可靠性测试宜在功能测试后进行

### （9）4类可靠性数据（高频）

- 失效时间数据
- 失效间隔时间数据
- 分组时间失效数
- 累积失效数
