# 9.3 软件可靠性管理

## 1. 本节核心思想（100字以内）

软件可靠性管理是软件工程管理的重要组成部分，其目标是在软件生命周期各阶段持续开展可靠性活动，通过目标制定、可靠性设计、测试、建模、评价和改进等手段，以有限成本达到预期可靠性目标，实现全过程可靠性保障。

---

## 2. 考试高频考点（尽量精简）

### 软件可靠性管理定义
- 目标：提高和保证软件可靠性。
- 特点：贯穿软件开发全过程。
- 内容：目标、计划、进度、任务、修正措施等管理活动。

### 各阶段主要活动

| 阶段 | 主要可靠性活动 |
|--------|--------|
| 需求分析 | 定目标、定标准、定计划 |
| 概要设计 | 定度量、定验收方案、可靠性设计 |
| 详细设计 | 可靠性预测、可靠性设计 |
| 编码 | 单元测试、排错 |
| 测试 | 建模、评价、集成/系统测试 |
| 实施 | 验收测试、调整模型、评价 |

### 高频对应关系
- 可靠性目标 → 需求分析阶段
- 可靠性度量 → 概要设计阶段
- 可靠性预测 → 详细设计阶段
- 单元测试 → 编码阶段
- 可靠性建模 → 测试阶段
- 验收测试 → 实施阶段

---

## 3. 记忆口诀（尽量精简）

### 生命周期口诀

> 需求定目标，概要定指标；  
> 详细做预测，编码找缺陷；  
> 测试建模型，实施修模型。

### 阶段关键词口诀

> 目标 → 指标 → 预测 → 排错 → 建模 → 修模
