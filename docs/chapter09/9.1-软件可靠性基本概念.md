# 9.1 软件可靠性基本概念

## 1. 本节核心思想（100字以内）

软件可靠性是指软件在规定条件、规定时间内完成规定功能的能力，本质是软件不发生失效的概率。通过可靠度、失效强度、MTTF、MTTR、MTBF等指标对可靠性进行量化，并通过可靠性测试获取数据、评估软件是否达到预期可靠性目标。

---

## 2. 考试高频考点（尽量精简）

### （1）软件可靠性定义
- 规定条件 + 规定时间 + 规定功能
- IEEE定义：软件在规定条件和时间内不引起系统失效的概率

### （2）软件与硬件可靠性的区别（重点）
1. 软件复杂性高，失效多由设计缺陷导致
2. 软件无物理退化
3. 软件具有唯一性，可完全复制
4. 软件版本更新快

### （3）三种时间概念
- 自然时间（日历时间）
- 运行时间
- 执行时间（CPU时间，最准确）

### （4）可靠性指标（重点）
- 失效概率：F(t)
- 可靠度：R(t)=1-F(t)
- 失效强度：λ(t)=F'(t)
- 平均失效前时间：MTTF
- 平均恢复时间：MTTR
- 平均故障间隔时间：MTBF=MTTF+MTTR

### （5）可靠性目标
- 可靠度
- 失效强度
- MTTF
- 失效严重程度类

### （6）可靠性测试分类（重点）
- 广义可靠性测试：目标→测试→建模→评价全过程
- 狭义可靠性测试：可靠性试验，收集可靠性数据

### （7）可靠性测试目的
- 发现缺陷
- 获取可靠性数据
- 验证是否达到可靠性目标
