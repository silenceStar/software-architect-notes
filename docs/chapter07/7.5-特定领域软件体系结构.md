# 7.5 特定领域软件体系结构（DSSA）

## 1. 本节核心思想（100字以内）
DSSA是在特定领域中，通过领域分析提取共性需求，设计通用参考架构，并沉淀可复用构件，从而支持该领域多个系统的快速构建与复用的软件体系结构方法。

---

## 2. 考试高频考点（尽量精简）
- DSSA定义：领域模型 + 参考需求 + 参考架构 + 可复用构件
- 两种领域划分：
  - 垂直域（完整系统族，行业级）
  - 水平域（跨系统功能）
- 三大阶段：
  - 领域分析（建模需求）
  - 领域设计（形成DSSA）
  - 领域实现（构件复用）
- 四类人员：
  - 领域专家
  - 领域分析人员
  - 领域设计人员
  - 领域实现人员
- DSSA建立过程五步：
  1. 定义领域范围
  2. 定义领域元素
  3. 定义约束
  4. 建立模型与架构
  5. 构建可复用构件
- 关键思想：复用 + 领域共性抽象 + 架构模板化
