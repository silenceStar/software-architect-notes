# 7.2 基于架构的软件开发方法（ABSD）总结

## 1. 本节核心思想（100字以内）
ABSD是一种以软件体系结构为核心驱动的开发方法，强调在需求未完全明确时即可启动架构设计，并通过功能分解、质量属性驱动和模板复用逐步细化系统结构，实现架构设计与需求分析并行迭代。

---

## 2. 考试高频考点（尽量精简）

### ⭐ ABSD三大基础
- 功能分解（模块拆分）
- 质量驱动架构风格选择（性能/可靠性/扩展性）
- 软件模板复用

### ⭐ 开发过程六阶段
- 体系结构需求
- 体系结构设计
- 文档化
- 复审
- 实现
- 演化

### ⭐ 关键概念
- 用例：功能需求
- 质量场景：非功能需求（性能/可靠性/变更等）
- 视图：逻辑 / 进程 / 实现 / 部署

### ⭐ 架构设计特点
- 递归 + 迭代 + 逐步细化
- 架构驱动开发（可早期开始设计）

### ⭐ 构件生成过程
- 类图生成 → 类分组 → 构件打包

### ⭐ 架构复审重点
- 是否满足需求
- 是否满足质量属性
- 构件划分是否合理

### ⭐ 架构演化
- 需求变化归类 → 计划 → 修改构件 → 更新关系 → 组装测试 → 评审

---

## 3. 记忆口诀（尽量精简）

### ⭐ ABSD三大基础
**“分功能，选风格，用模板”**

### ⭐ 六大流程
**“需设文审实演”**
（需求 → 设计 → 文档 → 复审 → 实现 → 演化）

### ⭐ 构件生成
**“类生图，组分类，打包成构件”**

### ⭐ 核心思想
**“架构先行，需求并行，递归细化”**
