# 7.3 软件架构风格

## 1. 本节核心思想（100字以内）
软件架构风格是对系统组织方式的抽象模板，通过定义构件、连接件及其约束，形成可复用的系统结构模式，用于指导不同类型软件系统的设计与重用，提升开发效率与系统一致性。

---

## 2. 考试高频考点（尽量精简）

### （1）数据流风格
- 批处理：阶段式顺序处理（ETL）
- 管道-过滤器：流式数据处理（Flink）

### （2）调用/返回风格
- 主程序/子程序：层级调用结构
- 面向对象：对象封装+方法调用
- 层次结构：严格分层（只能相邻层通信）
- C/S结构：两层/三层架构（瘦客户端/胖客户端）

### （3）数据中心风格
- 仓库：所有数据集中管理
- 黑板：黑板 + 知识源 + 控制模块 多知识源协同推理（AI/语音识别）

### （4）虚拟机风格
- 解释器：自定义语言执行（Python/SQL）
- 规则系统：if-then规则驱动（专家系统）

### （5）独立构件风格
- 进程通信：RPC/消息传递
- 事件系统：发布-订阅（Event-driven）
