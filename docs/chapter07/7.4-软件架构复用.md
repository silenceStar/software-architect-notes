# 7.4 软件架构复用

## 1. 本节核心思想（100字以内）
软件架构复用是通过产品线思想，将需求、架构、构件、测试、过程等软件资产进行系统化复用与管理，形成核心资产库，在不同产品中进行选择、裁剪与组装，从而提升开发效率、质量并降低成本。

---

## 2. 考试高频考点
- 软件产品线：共享核心资产，批量生成同类系统
- 复用类型：机会复用（边做边用） vs 系统复用（提前规划）
- 复用对象：需求、架构、构件、模型、测试、过程、人员等
- 核心资产库：构件存储、分类、检索、管理
- 构件库关键问题：分类与检索
- 复用流程：构造资产 → 管理资产 → 使用资产（选、改、组装）
- 复用目标：降低成本、缩短周期、提高质量
