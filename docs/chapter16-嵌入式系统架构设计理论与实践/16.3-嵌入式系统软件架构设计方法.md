# 16.3 嵌入式系统软件架构设计方法

## 1. 本节核心思想（100字以内）

本节介绍嵌入式系统常用的软件架构设计方法，包括 **ABSD（架构驱动设计）**、**ADD（属性驱动设计）** 和 **DARTS（实时系统设计）**。其中 ABSD 强调功能架构设计，ADD 强调质量属性驱动架构设计，DARTS 强调实时系统的并发任务划分，是考试重点。

---

## 2. 考试高频考点

### 架构设计方法关系图（建议画图，便于理解）

```text
              嵌入式软件架构设计
                      │
      ┌───────────────┼───────────────┐
      │               │               │
   ABSD           ADD             DARTS
（架构驱动）   （属性驱动）     （实时驱动）
      │               │               │
 功能需求优先    质量属性优先     并发任务优先
```

### 考点1：ABSD（Architecture-Based Software Design）

- **以功能需求和业务需求驱动架构设计**，采用**自顶向下、递归迭代**的方法，将系统逐步细化为模块。

### 考点2：ADD（Attribute-Driven Design）

- **以质量属性驱动架构设计**，围绕可靠性、安全性、性能、可维护性等质量属性进行架构设计，是本节重点。

### 考点3：质量场景六要素（★★★★★）

- ADD 使用质量场景描述质量需求，包括：
  - 刺激源（Source）
  - 刺激（Stimulus）
  - 环境（Environment）
  - 制品（Artifact）
  - 响应（Response）
  - 响应度量（Response Measure）

口诀：

> **源激环制响应量**

### 考点4：ADD 七步开发过程（★★★★★）

```text
评审输入
    ↓
确定驱动因子
    ↓
选择系统元素
    ↓
选择设计概念
    ↓
实例化元素、定义接口
    ↓
记录设计
    ↓
分析评估
```

核心思想：

> **质量属性 → 驱动架构设计 → 不断迭代完善**

### 考点5：DARTS（Design Approach for Real-Time System）

- **针对实时系统设计的方法**，核心思想是**将系统划分为多个并发任务，并设计任务之间的通信接口**。

适用于：

- 工业控制
- 航空航天
- 轨道交通
- 自动驾驶
- 机器人

### 考点6：DARTS 五步开发过程（★★★★★）

```text
RTSA需求分析
      ↓
划分并发任务
      ↓
定义任务接口
      ↓
设计各任务内部
      ↓
形成完整实时系统
```

重点：

- 并发任务
- 消息通信
- 事件同步
- 信息隐藏模块（IHM）

### 易混点

- **ABSD 与 ADD**
  - ABSD：关注**功能如何分解**。
  - ADD：关注**质量属性如何满足**。

- **ADD 与 DARTS**
  - ADD：适用于各种软件架构设计，以**质量属性**驱动。
  - DARTS：专门针对**实时系统**，以**并发任务划分**为核心。

### 易考选择题

- ABSD 的特点是（**自顶向下、递归迭代**）。
- ADD 的核心驱动因素是（**质量属性**）。
- 质量场景包括哪六部分（**刺激源、刺激、环境、制品、响应、响应度量**）。
- DARTS 最重要的思想是（**将实时系统划分为多个并发任务**）。
- DARTS 中任务间接口包括（**消息、事件、信息隐藏模块 IHM**）。

## 3. 记忆口诀

### 口诀一（三区分）

> **ABSD看功能，ADD看质量，DARTS看实时。**

解释：

- ABSD：功能驱动架构设计
- ADD：质量属性驱动架构设计
- DARTS：实时并发任务设计

---

### 口诀二（质量场景）

> **源激环制响应量**

解释：

- 源：刺激源
- 激：刺激
- 环：环境
- 制：制品（系统）
- 响：响应
- 量：响应度量

---

### 口诀三（ADD流程）

> **评驱元模接图评**

解释：

- 评：评审输入
- 驱：确定驱动因子
- 元：选择系统元素
- 模：选择设计概念（模式）
- 接：实例化元素、定义接口
- 图：记录设计视图
- 评：分析评估

---

### 口诀四（DARTS流程）

> **分析、分任务、连接口、做设计、并发运行。**

解释：

- RTSA 分析需求
- 划分并发任务
- 定义任务通信接口
- 设计各任务
- RTOS 并发调度运行

---

## 4. 考试一句话总结（精简）

> **本节最容易考：ABSD、ADD、DARTS 三种架构设计方法的区别，重点掌握 ADD 的质量场景六要素和 DARTS 的并发任务设计思想。**
