# 15.4 SOA 主要协议和规范

## 协议关系图

```text
                 服务提供者（Provider）
                        │
          ① 发布服务（WSDL 描述）
                        │
                        ▼
              ┌───────────────────┐
              │   UDDI 注册中心    │
              │（服务注册/发现）   │
              └───────────────────┘
                        ▲
          ② 查询服务（UDDI）
                        │
                 服务消费者（Consumer）
                        │
          ③ 根据WSDL了解接口
                        │
          ④ 使用SOAP调用服务
                        │
                        ▼
                 服务提供者（Provider）
```

> **现代微服务一般简化为：客户端 → REST API（HTTP+JSON）→ 服务。**

---

## 1. 本节核心思想（100字以内）

SOA通过一套标准协议实现服务注册、发现、描述和调用。传统Web Service主要采用 **UDDI+WSDL+SOAP**，分别负责服务发现、接口描述和消息通信；现代微服务则更多采用 **REST（HTTP+JSON）** 作为接口设计风格。

---

## 2. 考试高频考点

- **考点1：UDDI（服务注册中心）**
  - 用于服务注册、发布、发现，相当于"服务黄页"。

- **考点2：WSDL（服务描述语言）**
  - 使用XML描述Web Service接口，包括做什么、怎么调用、在哪里。

- **考点3：SOAP（通信协议）**
  - 基于XML的Web Service通信协议，负责服务之间消息传输。

- **考点4：REST（设计风格）**
  - 一种Web架构风格，以资源（Resource）为中心，通过HTTP方法操作资源，通常采用JSON数据格式。

- **易混点**
  - **REST不是协议，而是一种架构设计风格；SOAP才是通信协议。**

- **易考选择题**
  - **UDDI→服务注册发现；WSDL→接口描述；SOAP→消息通信；REST→资源风格（HTTP+JSON）。**

---

## 3. 记忆口诀

### 口诀

> **找UDDI，看WSDL，调SOAP，新项目REST跑。**

### 解释

- **找UDDI**：寻找服务（注册、发现）
- **看WSDL**：查看接口说明书
- **调SOAP**：发送XML消息调用服务
- **新项目REST跑**：现代微服务普遍采用REST API

---

## 4. 考试一句话总结（精简）

> **SOA经典三件套是 UDDI（发现）+ WSDL（描述）+ SOAP（通信），现代微服务主要采用 REST（HTTP+JSON）。**
