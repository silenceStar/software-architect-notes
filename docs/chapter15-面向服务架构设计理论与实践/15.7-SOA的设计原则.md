# 15.7 SOA 的设计原则

## SOA 八大设计原则

```text
                 SOA设计原则

                     SOA
                      │
 ┌────────────────────┼────────────────────┐
 │                    │                    │
无状态            单一实例          明确定义接口
 │                    │                    │
避免状态依赖      避免功能重复      WSDL公开接口
                      │
 ├────────────────────┼────────────────────┤
 │                    │                    │
自包含模块化       粗粒度            松耦合
 │                    │                    │
独立部署维护      服务少而大      只依赖接口
                      │
 ├────────────────────┼────────────────────┤
 │                    │
可重用            互操作兼容
 │                    │
重复利用        WS-Policy声明策略
```

---

## 1. 本节核心思想（100字以内）

SOA设计强调服务独立、标准化和松耦合，通过无状态、粗粒度、自包含、接口明确等原则，提高服务的可复用性、可维护性、可扩展性以及跨平台互操作能力。

---

## 2. 考试高频考点

- **考点1：SOA八大设计原则**（解释一句）

  无状态、单一实例、明确定义接口、自包含模块化、粗粒度、松耦合、可重用、互操作兼容，是SOA最经典的设计原则。

- **考点2：WSDL 与 WS-Policy 的作用**（解释一句）

  **WSDL**定义服务接口；**WS-Policy**定义服务策略（如安全、事务、服务等级等）。

- **易混点**（解释）

  **粗粒度 ≠ 功能简单。**  
  粗粒度表示一个服务完成一项完整业务，而不是把一个业务拆成很多细小接口，从而减少服务间通信次数。

- **易考选择题**（解释）

  **松耦合**表示客户端只依赖服务接口，不关心服务的位置、实现技术和内部数据，是SOA最重要的设计思想之一。

---

## 3. 记忆口诀

**口诀：**

> **无单接口，自粗松；重互策略全兼容。**

**解释：**

- **无**：无状态
- **单**：单一实例
- **接口**：明确定义接口（WSDL）
- **自**：自包含、模块化
- **粗**：粗粒度
- **松**：松耦合
- **重**：重用能力
- **互**：互操作与兼容（WS-Policy）

---

## 4. 考试一句话总结（精简）

**SOA最重要的设计思想是：服务标准化、松耦合、自包含、可重用，其中WSDL定义接口，WS-Policy定义策略，是考试高频考点。**
