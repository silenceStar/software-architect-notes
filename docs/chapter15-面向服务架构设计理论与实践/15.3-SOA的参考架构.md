# 15.3 SOA 的参考架构
```text
                     用户
                      │
               交互服务（页面）
                      │
                流程服务（BPEL）
                      │
         ----------------------------
         │                          │
 信息服务（数据）           业务逻辑服务（业务）
         │                          │
         └────────ESB──────────────┘
                      │
              IT基础设施、安全
```
## 1. 本节核心思想（100字以内）

SOA参考架构以**服务**为中心，通过**业务逻辑、控制、连接（ESB）、开发、业务优化、IT管理**六大服务，实现企业内部和外部系统的松耦合集成。其中**ESB（企业服务总线）**是SOA架构的核心，负责服务之间的连接、转换、路由与解耦。

---

## 2. 考试高频考点

- **考点1：SOA六大服务**
  - 业务逻辑服务、控制服务、连接服务（ESB）、开发服务、业务创新与优化、IT服务管理，各自职责需掌握。

- **考点2：ESB（企业服务总线）**
  - ESB是SOA核心，负责服务注册、消息转换、协议转换、路由、服务发现、负载均衡，实现服务松耦合。

- **考点3：控制服务组成**
  - 控制服务包括**信息服务（数据整合）**、**流程服务（业务编排）**、**交互服务（用户访问）**。

- **考点4：流程服务**
  - 负责业务流程编排（BPEL）、事务管理、补偿事务、人工审批，是SOA流程自动化的核心。

- **考点5：业务创新与优化**
  - 基于BPM，通过事件采集、KPI监控和业务分析，不断优化业务流程。

- **易混点**
  - **业务逻辑服务**负责"真正执行业务"；**控制服务**负责"组织业务流程"；**连接服务（ESB）**负责"连接各业务服务"。

- **易考选择题**
  - - ESB属于哪类服务？（连接服务）
    - 数据聚合属于哪类服务？（信息服务）
    - BPEL属于哪类服务？（流程服务）
    - SSO属于哪类服务？（交互服务/IT服务管理均可能出现，教材中交互服务负责用户体验，IT服务管理负责安全认证）
    - KPI监控属于哪类服务？（业务创新与优化）

## 3. 记忆口诀

### 口诀

> **业控连，优开管；控三件，数流人；ESB：转路连解。**

### 解释

- **业**——业务逻辑服务（真正干业务）
- **控**——控制服务（组织业务）
- **连**——连接服务（ESB）
- **优**——业务创新与优化（KPI、BPM）
- **开**——开发服务（建模、设计、测试）
- **管**——IT服务管理（安全、运维）

控制服务包括三部分：

- **数**——信息服务（数据整合）
- **流**——流程服务（流程编排、事务）
- **人**——交互服务（门户、SSO、多终端）

ESB四大能力：

- **转**——协议转换、消息转换
- **路**——消息路由、服务发现
- **连**——连接各种异构系统
- **解**——解耦服务请求者和提供者

---

## 4. 考试一句话总结（精简）

**SOA参考架构重点考六大服务，其中ESB负责连接与解耦，控制服务负责数据、流程和用户交互，是软考SOA章节的核心考点。**
