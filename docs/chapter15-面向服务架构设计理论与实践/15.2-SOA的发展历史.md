# 15.2 SOA 的发展历史

## 1. 本节核心思想（100字以内）

SOA经历了XML奠基、Web Service标准化、SOA成熟应用三个发展阶段，并逐渐演化为微服务架构。考试重点掌握SOA的发展历程、Web Service三大标准、国内外SOA发展差异，以及SOA与微服务的区别。

---

## 2. 考试高频考点

- **考点1：SOA发展三阶段（★★★★★）**
  - **萌芽阶段**：XML奠基数据交换标准。
  - **标准化阶段**：SOAP、WSDL、UDDI三大标准出现。
  - **成熟阶段**：SCA、SDO、WS-Policy推动SOA落地。

- **考点2：XML的作用**
  - SOA的数据交换基础，统一数据格式，实现跨平台通信。

- **考点3：Web Service三剑客（★★★★★）**
  - **SOAP**：通信协议。
  - **WSDL**：服务描述语言。
  - **UDDI**：服务注册与发现。

- **考点4：成熟阶段三大规范**
  - **SCA**：服务组件架构。
  - **SDO**：服务数据对象。
  - **WS-Policy**：服务安全与策略规范。

- **考点5：国内外SOA区别**
  - **美国**：改造遗留系统，封装已有服务。
  - **中国**：大量新建服务，同时逐步整合旧系统。

- **考点6：SOA与微服务区别（★★★★★）**
  | SOA | 微服务 |
  |------|---------|
  | 服务粒度较大 | 服务粒度更小 |
  | 依赖ESB | 去中心化，无ESB |
  | 部署耦合 | 独立部署 |
  | 集中式治理 | 分布式治理 |
  | 常用于企业集成 | 更适合互联网高并发 |

- **易混点**
  - **SOA不是微服务。**
  - **微服务来源于SOA，是SOA思想的进一步细化和演进。**

- **易考选择题**
  - XML是SOA发展的基础。
  - SOAP、WSDL、UDDI被称为**Web Service三剑客**。
  - 微服务最大的特点：**去中心化、独立部署、细粒度**。
  - SOA通常依赖**ESB企业服务总线**。

---

## 3. 记忆口诀

### 口诀

> **XML打基础，SOAP来通信；WSDL写说明，UDDI找服务；SOA靠ESB，微服务去中心。**

### 解释

- XML：统一数据格式。
- SOAP：负责通信。
- WSDL：描述服务接口。
- UDDI：注册和发现服务。
- SOA：集中式ESB。
- 微服务：去ESB、独立部署。

---

## 4. 考试一句话总结（精简）

**SOA经历XML→Web Service→成熟应用三个阶段，并最终演进为去中心化、细粒度的微服务架构。**
