# 15.5 SOA设计的标准要求

## 整体结构图

```text
                 SOA设计标准
                      │
 ┌──────────────┬──────────────┬──────────────┬────────────────────┐
 │              │              │              │
文档标准化    通信协议标准   服务注册与发现    服务质量（QoS）
(WSDL)       (XML/XSD/SOAP)    (UDDI)              │
                                                    │
                    ┌──────────┬──────────┬──────────┬──────────┬──────────┐
                    │          │          │          │          │
                 可靠性      安全性      策略      控制      管理
           WS-Reliable   WS-Security  WS-Policy  BPEL    WSDM
```

---

## 1. 本节核心思想（100字以内）

SOA要实现跨平台、跨语言、跨系统的服务协作，必须遵循统一标准，包括服务描述、消息通信、服务注册以及服务质量（QoS）等规范，使不同服务能够安全、可靠、可管理地进行交互。

---

## 2. 考试高频考点

- **考点1：SOA四大标准要求**
  - 文档标准（WSDL）、通信标准（XML/XSD/SOAP）、服务注册（UDDI）、服务质量（QoS）。

- **考点2：QoS包含哪些内容**
  - QoS主要包括可靠性、安全性、策略、控制、管理五个方面，并分别对应不同WS-*标准。

- **易混点**
  - **WSDL**负责**描述服务**；**UDDI**负责**注册和发现服务**；**SOAP**负责**消息通信**，三者职责不同。

- **易考选择题**
  - **WSDL → 服务描述**
  - **SOAP → 消息通信**
  - **UDDI → 服务注册与发现**
  - **WS-Policy → 服务策略**
  - **BPEL（WSBPEL）→ 服务流程编排**
  - **WSDM → 服务管理**
  - **WS-ReliableMessaging → 消息可靠传输**
  - **WS-Security → 服务安全**

---

## 3. 记忆口诀

**口诀：**

> **文通注质，质有五保。**

**解释：**

- **文**——WSDL（文档描述）
- **通**——SOAP/XML/XSD（通信标准）
- **注**——UDDI（注册发现）
- **质**——QoS（服务质量）
  - **保可靠**（Reliable Messaging）
  - **保安全**（WS-Security）
  - **保策略**（WS-Policy）
  - **保流程**（BPEL）
  - **保管理**（WSDM）
