# 13.7 层次式架构案例分析

## 1. 本节核心思想（100字以内）

通过 **PetShop** 和 **电子小票物联网** 两个案例，说明层次式架构在实际项目中的应用。重点掌握 **标准三层架构、面向接口设计、工厂模式、缓存、异步处理** 等思想，以及物联网三层架构（感知层、网络层、应用层）的职责划分。

---

## 2. 考试高频考点（精简）

- **考点1：PetShop架构演进**（2.0无DAL→3.0增加DAL→4.0引入接口、工厂、缓存和异步，提高扩展性。）
- **考点2：IDAL+DALFactory**（面向接口编程，实现数据库解耦，方便SQL Server与Oracle切换。）
- **考点3：Cache与MSMQ**（缓存提高查询性能，消息队列实现异步订单处理，提高响应速度。）
- **考点4：物联网三层架构**（感知层采集、网络层传输处理、应用层提供业务服务。）

- **易混点**
  - DAL负责数据库访问，不负责业务逻辑；BLL负责业务逻辑，不直接访问数据库。
  - IDAL是接口，DALFactory负责创建具体DAL实现，两者作用不同。

- **易考选择题**
  - PetShop 2.0 的主要缺陷是什么？
  - PetShop 4.0 为什么能够方便切换数据库？
  - Cache、MSMQ分别解决什么问题？
  - 电子小票系统中，Wi-Fi/GPRS属于哪一层？

---

## 3. 记忆口诀（精简）

**口诀：**
> **二乱三分四优化，接口工厂解耦佳；缓存异步提性能，物联三层采传用。**

**解释：**
- 二乱：PetShop2.0职责混乱。
- 三分：PetShop3.0增加DAL形成三层架构。
- 四优化：PetShop4.0加入IDAL、工厂、缓存、异步。
- 采传用：物联网三层分别是感知采集、网络传输、应用服务。

---

## 4. 考试一句话总结（精简）

**最容易考：PetShop 4.0 如何通过 IDAL、DALFactory、Cache、异步机制实现低耦合、高扩展、高性能的层次式架构。**
