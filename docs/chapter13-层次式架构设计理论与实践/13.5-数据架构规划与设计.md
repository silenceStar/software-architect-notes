# 13.5 数据架构规划与设计

## 1. 本节核心思想（100字以内）

数据架构设计应与面向对象类设计保持一致，以降低系统全生命周期成本；同时根据 XML 数据特点选择合适的存储方式（文件或数据库），兼顾数据管理、查询效率和系统性能。

---

## 2. 考试高频考点（精简）

- **考点1：数据库设计与类设计融合**（类对应实体，建模没有唯一正确答案，应关注系统生命周期成本。）
- **考点2：XML 文档分类**（分为数据中心型和文档中心型，区别在于结构规则性和是否关注元素顺序。）
- **考点3：XML 存储方式**（文件存储适合简单保存；数据库存储查询效率高、支持事务和并发。）
- **考点4：XML 与数据库关系**（XML 不能完全替代数据库，仅适用于数据量小、用户少、性能要求低的场景。）

**易混点**

- 数据中心型 XML（规则结构、关注数据） ≠ 文档中心型 XML（结构灵活、关注内容和顺序）。
- XML 文档 ≠ XML 数据库，XML 数据库有专门 DBMS 管理。

**易考选择题**

- 好的数据模型目标是什么？
- XML 文档有哪些类型？
- XML 有哪两种存储方式？各自优缺点？
- XML 能否完全替代传统数据库？

---

## 3. 记忆口诀（精简）

**口诀**

> **类映实体，建模无唯一；数据规则文档乱；文件易存库易查。**

**解释**

- **类映实体**：类设计应对应数据库实体设计。
- **建模无唯一**：好的模型关注全生命周期成本。
- **数据规则文档乱**：数据中心型结构规则，文档中心型结构灵活。
- **文件易存库易查**：文件存储简单，数据库存储查询和管理能力更强。

---

## 4. 考试一句话总结（精简）

**重点考数据库与类设计融合、XML 文档分类及 XML 两种存储方式的特点与适用场景。**
