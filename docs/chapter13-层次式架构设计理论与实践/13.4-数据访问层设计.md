# 13.4 数据访问层设计

## 1. 本节核心思想（100字以内）

数据访问层（DAL）负责业务系统与数据库之间的数据交互，通过 DAO、DTO、ORM、工厂模式、事务管理和连接池等技术，实现数据访问解耦、数据库无关性、事务一致性及高性能访问，提高系统的可维护性、可扩展性和开发效率。

---

## 2. 考试高频考点（精简）

- **考点1：五种数据访问模式**（在线访问、DAO、DTO、离线数据、ORM，理解作用及区别。★★★★★）
- **考点2：DAO 模式**（封装数据库访问，实现业务逻辑与数据访问分离。★★★★★）
- **考点3：DTO 模式**（封装数据传输对象，减少远程调用，不包含业务逻辑。★★★★★）
- **考点4：ORM 与 Hibernate**（对象与关系数据库映射，Hibernate 是典型 ORM 框架。★★★★★）
- **考点5：工厂模式**（根据数据库类型创建对应的数据访问对象，实现数据库无关。★★★★☆）
- **考点6：事务 ACID**（原子性、一致性、隔离性、持久性，历年高频。★★★★★）
- **考点7：数据库连接池**（连接复用，提高性能，避免频繁创建连接。★★★★☆）
- **考点8：XML Schema**（约束 XML 结构，支持丰富数据类型、继承、命名空间。★★☆☆☆）

### 易混点

- **DAO（访问数据库）≠ DTO（传输数据）**
- **ORM（自动映射对象）≠ DAO（封装数据库操作）**
- **JDBC 事务（单数据库）≠ JTA 事务（分布式事务）**
- **连接池归还连接，不是关闭连接。**

### 易考选择题

- DAO 的作用是什么？
- DTO 是否允许包含业务逻辑？（**不能**）
- ORM 的主要作用是什么？

- ACID 四个特性分别是什么？
- 工厂模式解决什么问题？（**数据库无关性**）
- 连接池为什么能提升性能？

## 3. 记忆口诀（精简）

### 口诀

**在线DAO传DTO，离线ORM映对象；工厂切库连池快，事务牢记原一隔持。**

### 解释

- **在线**：直接访问数据库
- **DAO**：封装数据库访问
- **DTO**：封装数据传输
- **离线**：脱离数据库连接操作数据
- **ORM**：对象与数据库自动映射
- **工厂**：屏蔽数据库差异
- **连接池**：连接复用，提高性能
- **原一隔持**：ACID（原子、一致、隔离、持久）
