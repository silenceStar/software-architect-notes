# 13.1 层次式体系结构概述

## 1. 本节核心思想（100字以内）

层次式体系结构（N层架构）按照职责将系统划分为多个层次，各层只负责本层功能，并通过相邻层交互，实现高内聚、低耦合，提高系统的可维护性、可扩展性和复用性，是企业应用最常用的软件架构模式。

---

## 2. 考试高频考点（精简）

- **考点1：层次式架构思想**（按职责分层，上层调用下层，相邻层通信。）
- **考点2：四层架构**（表现层→业务层→数据访问层→数据层，各层职责要会区分。）
- **考点3：层次式架构优点**（结构清晰、易维护、易开发、易测试、易复用。）
- **考点4：层次式架构缺点**（系统容易臃肿、调用链变长、性能下降。）
- **考点5：污水池反模式**（请求层层穿透，中间层几乎没有业务逻辑，是考试高频概念。）

### 易混点
- **业务层**负责业务规则，不负责数据库操作。
- **数据访问层（DAO）**负责数据库访问，不负责业务逻辑。
- **数据层**是数据库本身，不等于DAO。

### 易考选择题
- 层次式架构遵循**相邻层通信**原则。
- Java EE/Spring 等企业应用普遍采用**N层架构**。
- **污水池反模式**的特点及解决思路（开放部分层）。

---

## 3. 记忆口诀（精简）

**口诀：**
> **四层分工，相邻通信；高内聚低耦合；污水池要避免。**

**解释：**
- **四层分工**：表现→业务→访问→数据。
- **相邻通信**：只能调用下一层，不能跨层访问。
- **高内聚低耦合**：层内职责集中，层间依赖最小。
- **污水池避免**：中间层不能只是"传话筒"。

---

## 4. 考试一句话总结（精简）

**层次式架构最常考四层职责、优缺点以及"污水池反模式"的概念，是选择题高频考点。**
