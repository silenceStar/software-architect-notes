# 2.6 计算机语言

## 1. 本节核心思想（100字以内）

计算机语言是人与计算机交换信息的工具，主要包含表达式、流程控制和集合。按用途和抽象程度可分为机器语言、汇编语言、高级语言、建模语言和形式化语言，各类语言在执行效率、可移植性、表达能力和使用场景上各有特点。

> 本节以概念分类和对比为主，不画图也能理解；记忆时可抓住“机器执行、汇编符号、高级编程、UML建模、形式化验证”这条主线。

## 2. 考试高频考点

- **考点1：机器语言、汇编语言和高级语言的区别**  
  机器语言由二进制指令组成，可被CPU直接执行；汇编语言是机器指令的符号化表示，需要汇编程序翻译；高级语言更接近自然语言，易于开发且可移植性较好。

- **考点2：机器指令的组成和地址格式**  
  一条机器指令通常由操作码和地址码组成，按显式地址数量可分为三地址、二地址、单地址和零地址指令，其中零地址指令常见于堆栈型计算机。

- **考点3：汇编语言的三种语句**  
  汇编语言包括指令语句、伪指令语句和宏指令语句；指令语句在运行时由CPU执行，伪指令在汇编时处理且通常不生成机器代码，宏指令是对已定义程序段的引用。

- **考点4：UML的组成与基本关系**  
  UML由基本构造块、图和公用机制组成；基本构造块包括事物和关系，其中四种基本关系是依赖、关联、泛化和实现。

- **考点5：UML的图和五种视图**  
  UML 2.0包括类图、用例图、序列图、活动图、状态图、构件图和部署图等13种图；五种视图为用例视图、逻辑视图、进程视图、实现视图和部署视图，用例视图居于中心。

- **考点6：形式化语言和Z语言**  
  形式化方法使用具有精确数学语义的语言描述、分析和验证软件系统；Z语言以集合论和一阶谓词逻辑为基础，使用模式描述系统状态和操作。

- **易混点：指令语句与伪指令语句**  
  指令语句汇编后产生机器代码，其操作在程序运行时完成；伪指令通常不产生机器代码，其操作在源程序汇编时完成。

- **易混点：泛化与实现**  
  泛化表示子类继承父类，是“特殊—一般”关系，通常画空心三角实线；实现表示类或构件履行接口规定的契约，通常画空心三角虚线。

- **易混点：UML是语言而不是开发方法**  
  UML是一种与具体编程语言无关的标准建模语言，只规定模型的表示符号和规则，本身不规定完整的软件开发过程。

- **易考选择题：哪种语言能被计算机直接执行？**  
  机器语言可以被CPU直接识别和执行，汇编语言和高级语言都必须经过相应的翻译处理。

- **易考选择题：哪种汇编语句不产生机器代码？**  
  伪指令语句通常不产生机器代码，它主要用于指导汇编程序分配存储空间、定义变量或完成其他汇编工作。

- **易考选择题：UML中表示系统外部可见功能的是哪种图？**  
  用例图从外部使用者角度描述系统提供的功能，适合对系统语境和功能需求进行建模。

- **易考选择题：哪种形式化语言采用“状态—操作”风格？**  
  Z语言采用“状态—操作”风格，以集合论和一阶谓词逻辑作为形式语义基础。

## 3. 记忆口诀

### 口诀

**机汇高，模形全；操地组成指令篇；指伪宏，运汇分；事关图，依关泛实；用逻进实部；Z集谓，状态操作保精确。**

### 解释

- **机汇高，模形全**：计算机语言包括机器语言、汇编语言、高级语言、建模语言和形式化语言。
- **操地组成指令篇**：机器指令由操作码和地址码组成。
- **指伪宏，运汇分**：汇编语句分为指令、伪指令和宏指令；指令在运行时执行，伪指令在汇编时处理。
- **事关图，依关泛实**：UML基本内容包括事物、关系和图；四种关系是依赖、关联、泛化和实现。
- **用逻进实部**：UML五种视图是用例、逻辑、进程、实现和部署视图。
- **Z集谓，状态操作保精确**：Z语言基于集合论和一阶谓词逻辑，通过状态和操作精确描述系统。

本节最可能考机器语言、汇编语言和高级语言的区别，汇编语句类型，UML四种关系与五种视图，以及Z语言的特点。
