# 2.5 计算机网络

## 1. 本节核心思想（100字以内）

计算机网络通过通信线路、网络设备和通信协议，将地理上分散的计算机连接起来，实现数据通信、资源共享、集中管理、分布式处理和负载均衡。学习重点是网络性能指标、通信技术、网络分类、组网设备、OSI与TCP/IP模型，以及交换和路由原理。

## 2. 考试高频考点

- **考点1：网络性能指标**  
  速率表示数据传输速度，带宽表示线路的最大传输能力，吞吐量表示实际通过的数据量，时延表示数据从一端到另一端所需时间，RTT表示一次发送与确认的往返时间。

- **考点2：网络设备及工作层次**  
  中继器、集线器工作在物理层，网桥、交换机工作在数据链路层，路由器工作在网络层，防火墙依据安全规则监视和过滤进出网络的数据。

- **考点3：OSI与TCP/IP模型**  
  OSI模型分为物理层、数据链路层、网络层、传输层、会话层、表示层和应用层；TCP/IP模型分为网络接口层、网际层、传输层和应用层。

- **考点4：局域网拓扑结构**  
  星状结构便于管理但中心结点故障会导致全网瘫痪；总线结构成本低但总线故障影响全网；环形结构无路径选择但扩展困难；网状结构可靠性高但建设成本高。

- **考点5：交换机转发原理**  
  交换机根据数据帧的源MAC地址学习端口映射，根据目的MAC地址进行转发；目的地址未知时执行泛洪，MAC地址表会定期更新。

- **考点6：路由器的主要功能**  
  路由器根据路由表在不同网络间选择数据转发路径，还可以完成异种网络互连、协议转换、速率适配、网络隔离、报文分片与流量控制。

- **易混点：带宽、速率和吞吐量**  
  带宽侧重线路理论上的最大数据传输能力，速率侧重设备实际发送数据的快慢，吞吐量是单位时间内实际成功通过网络的数据量，通常不超过带宽。

- **易混点：复用技术与多址技术**  
  复用技术解决一条信道同时传输多路数据的问题，多址技术解决多个用户共同使用信道并在接收端区分用户的问题，复用技术是多址技术的基础。

- **易考选择题：设备与OSI层次对应关系**  
  集线器和中继器对应物理层，网桥和交换机对应数据链路层，路由器对应网络层；交换机根据MAC地址转发，路由器根据IP地址和路由表转发。

- **易考选择题：5G三类典型应用场景**  
  eMBB强调高速率和大带宽，适合超高清视频、AR和VR；mMTC强调海量设备连接，适合物联网；uRLLC强调超低时延和高可靠性，适合自动驾驶、工业控制和远程医疗。

## 3. 记忆口诀

### 口诀一：网络五大功能

**“通、享、管、分、均”**

解释：

- 通：数据通信
- 享：资源共享
- 管：集中管理
- 分：分布式处理
- 均：负荷均衡

### 口诀二：网络性能指标

**“速带吞延，往返利用”**

解释：

- 速：速率
- 带：带宽
- 吞：吞吐量
- 延：时延
- 往返：往返时间RTT
- 利用：信道利用率和网络利用率

### 口诀三：OSI七层

**“物数网传会表应”**

解释：

- 物：物理层
- 数：数据链路层
- 网：网络层
- 传：传输层
- 会：会话层
- 表：表示层
- 应：应用层

### 口诀四：网络设备工作层次

**“物理集中继，链路桥交换，网络找路由”**

解释：

- 集线器、中继器工作在物理层
- 网桥、交换机工作在数据链路层
- 路由器工作在网络层

### 口诀五：交换机工作原理

**“源址学习，目的转发；地址未知，广播泛洪”**

解释：

交换机从源MAC地址学习端口映射，根据目的MAC地址查询转发端口；查询不到目的MAC地址时，向除源端口以外的其他端口泛洪。

## 4. 考试一句话总结（精简）

本节最可能考网络性能指标的区别、OSI与TCP/IP分层对应关系、网络设备工作层次，以及交换机和路由器的转发原理。
