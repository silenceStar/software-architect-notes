# 2.3 计算机软件

## 1. 本节核心思想（100字以内）

计算机软件是程序、数据和相关文档的集合，分为系统软件和应用软件。系统软件负责管理硬件资源、提供公共运行环境；应用软件解决具体业务问题。本节重点掌握操作系统、数据库、文件系统、中间件和软件构件的基本概念、分类及特点。

```text
用户
  ↓
应用软件
  ↓
中间件、软件构件
  ↓
操作系统、数据库管理系统、网络软件
  ↓
计算机硬件
```

## 2. 考试高频考点

* **考点1：系统软件与应用软件**
  系统软件管理计算机资源并为其他软件提供服务，如操作系统、数据库管理系统和编译程序；应用软件用于解决特定业务问题，如财务软件、办公软件和游戏软件。

* **考点2：操作系统的功能与特征**
  操作系统是计算机资源的管理者，负责处理器、存储、文件和设备管理，其基本特征是并发性、共享性、虚拟性和不确定性。

* **考点3：操作系统的分类**
  批处理系统强调提高资源利用率，分时系统强调多用户交互和及时响应，实时系统强调在规定时间内完成处理，分布式系统强调多机协同和透明访问。

* **考点4：数据库及数据库设计**
  数据模型三要素是数据结构、数据操作和完整性约束；数据库设计依次包括需求分析、概念结构设计、逻辑结构设计、物理结构设计、应用程序设计和运行维护。

* **考点5：文件系统及文件结构**
  文件系统负责文件的统一管理、按名存取、权限控制和空间管理；文件的物理结构主要有连续结构、链接结构和索引结构。

* **考点6：中间件与软件构件**
  中间件位于应用软件和操作系统、网络、数据库之间，用于屏蔽底层差异；软件构件具有自包容、可复用和通过接口访问等特点。

* **易混点：并发与并行**
  并发是多个任务在一段时间内交替执行，单CPU下同一时刻通常只有一个任务运行；并行是多个CPU或核心在同一时刻真正同时执行多个任务。

* **易混点：分时系统与实时系统**
  分时系统追求多个用户都能获得及时交互响应；实时系统强调任务必须在规定的截止时间内完成，重点是时间约束和可靠性。

* **易混点：概念、逻辑和物理结构设计**
  概念结构设计主要建立E-R模型；逻辑结构设计将E-R模型转换为关系表并确定约束；物理结构设计确定索引、分区、存储结构和访问路径。

* **易混点：网络操作系统与分布式操作系统**
  网络操作系统主要实现多台独立计算机之间的资源共享；分布式操作系统将多台计算机统一管理，对用户表现为一个整体系统。

* **易混点：硬链接与符号链接**
  硬链接是多个文件名指向同一个文件实体；符号链接是一个保存目标文件路径的新文件，可以跨文件系统，但目标删除后链接会失效。

* **易考选择题：典型产品与类型对应**
  IBM MQSeries属于消息中间件，BEA Tuxedo属于事务处理中间件；CORBA由OMG提出，J2EE基于Java平台，DNA属于Microsoft分布式构件体系。

## 3. 记忆口诀

### 口诀

> 软件程序数文档，系统应用分两旁；
> 操作系统管资源，并共虚不四特点；
> 批分实时网分布，嵌入微小又可靠；
> 数据模型三要素，需概逻物应维护；
> 文件连续链索引，中间件在上下间；
> 构件自包可复用，接口组装成系统。

### 解释

* **程序数文档**：软件由程序、数据和相关文档组成。
* **系统应用分两旁**：软件分为系统软件和应用软件。
* **并共虚不**：操作系统的四个特征是并发、共享、虚拟和不确定。
* **批分实时网分布**：操作系统包括批处理、分时、实时、网络和分布式等类型。
* **需概逻物应维护**：数据库设计顺序为需求、概念、逻辑、物理、应用和运行维护。
* **连续链索引**：文件的主要物理结构是连续、链接和索引结构。
* **中间件在上下间**：中间件位于应用软件和底层系统软件之间。
* **自包可复用**：软件构件最重要的特点是自包容和可复用。

本节最可能考软件分类、操作系统的四大特征及类型、数据库设计阶段、文件结构，以及中间件和软件构件的特点。
