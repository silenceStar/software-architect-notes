# 17.4 网络构建和设计方法

## 1. 本节核心思想（100字以内）

网络构建应先完成业务、用户、应用、平台和网络需求分析，再进行局域网、广域网、地址、路由及分层架构设计，并通过冗余、安全控制和绿色技术，在性能、可靠性、可用性、成本与节能之间取得平衡。

### 网络设计总体流程

```text
需求分析
   ↓
技术选型
   ↓
地址与路由规划
   ↓
层次化网络设计
   ↓
高可用与安全设计
   ↓
绿色网络设计
```

## 2. 考试高频考点

* **考点1：网络需求分析的五个方面**
  包括业务需求、用户需求、应用需求、计算机平台需求和网络需求，最终形成网络需求规格说明书。

* **考点2：局域网与广域网技术选择**
  局域网重点掌握 STP 防环路、VLAN 划分广播域、链路与服务器冗余；广域网重点掌握远程接入、VPN、专线及性能优化方法。

* **考点3：三层层次化网络模型**
  核心层负责高速转发和高可用，汇聚层负责路由汇总、访问控制及流量策略，接入层负责终端接入、用户认证和信息收集。

```text
              核心层
       高速转发、高可靠、少策略
                 │
              汇聚层
       路由汇总、访问控制、流量策略
                 │
              接入层
       用户接入、认证、地址与日志采集
```

* **考点4：网络高可用设计**
  提高可用性有两条主要途径：提高网络可靠性、缩短故障恢复时间；冗余应部署在关键节点和链路，不能无限叠加。

* **考点5：网络安全技术**
  主要包括防火墙、VPN、访问控制、安全隔离、安全协议和安全审计；HTTPS 本质上是基于 SSL/TLS 的安全 HTTP 通信。

* **考点6：绿色网络设计原则**
  绿色网络不仅关注设备功耗，还应从标准化、集成化、虚拟化和智能化方面优化整体架构，降低设备数量、能源消耗和总体拥有成本。

* **易混点：可靠性与可用性**
  可靠性强调网络连续无故障运行的能力；可用性强调一定时间内网络可以正常使用的比例。可靠性高但恢复时间长，可用性仍可能较低。

* **易混点：线路备份与负载分担**
  备份方式通常只有主链路传输，备用链路故障时启用；负载分担方式多条链路同时承载业务，资源利用率更高。

* **易混点：核心层与汇聚层**
  核心层追求高速、简单和稳定，不宜配置复杂策略；访问控制、路由汇总和流量管理通常放在汇聚层。

* **易考选择题：网络设计从哪一层开始？**
  应先从接入层分析用户数量、流量和访问行为，再自下而上完成汇聚层与核心层的容量规划。

* **易考选择题：STP 和 VLAN 分别解决什么问题？**
  STP 用于防止二层交换网络环路；VLAN 用于划分广播域，实现逻辑隔离。

* **易考选择题：服务器高可用有哪些实现方式？**
  包括负载均衡器、NAT 地址转换、DNS 轮询及双机热备；双机热备又分为单活和双活模式。

## 3. 记忆口诀

### 口诀一：需求分析

> **业务用户看应用，平台网络成规范。**

解释：

* 业务需求：明确建设目标、预算及发展趋势；
* 用户需求：明确用户需要的服务质量；
* 应用需求：明确带宽、响应时间和可靠性；
* 平台需求：明确接入的设备类型；
* 网络需求：明确拓扑、性能、安全和管理要求；
* 最终形成网络需求规格说明书。

### 口诀二：三层网络

> **核心快，汇聚管，接入接。**

解释：

* 核心层：高速转发，保证骨干网络可靠；
* 汇聚层：实施路由、安全和流量策略；
* 接入层：连接用户、终端、AP及其他设备。

### 口诀三：局域网设计

> **生成树防环，VLAN隔广播；链路做冗余，服务器保连续。**

解释：

* STP 防止二层网络环路；
* VLAN 划分广播域；
* 冗余链路提高传输可用性；
* 负载均衡和双机热备保证服务器连续服务。

### 口诀四：高可用设计

> **少故障，快恢复；关键冗余，不过度。**

解释：

* 提高可靠性，减少故障发生；
* 缩短恢复时间，降低故障影响；
* 对关键节点和链路进行冗余；
* 避免无限堆叠设备导致成本和复杂度过高。

### 口诀五：网络安全

> **墙、隧、控、隔、议、审。**

解释：

* 墙：防火墙；
* 隧：VPN 加密隧道；
* 控：访问控制；
* 隔：网络安全隔离；
* 议：SSL、TLS、HTTPS 等安全协议；
* 审：网络安全审计。

### 口诀六：绿色网络

> **标、集、虚、智，安全可靠不能失。**

解释：

* 标准化：减少异构设备和转换成本；
* 集成化：减少设备、线缆和空间占用；
* 虚拟化：提高资源利用率；
* 智能化：实现统一管理、故障诊断和自动恢复；
* 同时还要兼顾安全性与可靠性。
