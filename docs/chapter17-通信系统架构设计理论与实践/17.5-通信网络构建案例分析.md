# 17.5 通信网络构建案例分析

## 1. 本节核心思想

通过高可用分层组网、IPv4/IPv6双栈演进及5G行业应用案例，说明通信网络应综合考虑设备与链路冗余、故障快速恢复、平滑升级、地址规划，以及高带宽、低时延和高可靠等业务需求。

---

## 2. 考试高频考点

* **考点1：高可用网络分层设计**
  接入层通过双归属、链路聚合、MSTP和VRRP实现冗余；汇聚层通过OSPF、双机FW/LB实现故障切换；核心层采用双核心、三层互联和ECMP提高可靠性与带宽利用率。

* **考点2：IPv4/IPv6双栈园区网构建**
  核心层和汇聚层优先升级为双栈，接入层可保留原有二层设备；不具备双栈条件的区域可通过ISATAP等隧道技术接入IPv6网络。

* **易混点：MSTP、VRRP、OSPF的作用不同**
  MSTP用于消除二层环路，VRRP用于实现默认网关或设备主备切换，OSPF用于三层网络的动态选路与故障收敛，三者不能相互替代。

* **易考选择题：典型技术与应用对应关系**
  接入—汇聚高可用优先采用三角形组网；双栈升级遵循设备评估、业务评估、地址规划和方案实施；5G智能电网通过MEC与uRLLC满足低于8ms的低时延、高可靠通信需求。

### 高可用网络关系图

```text
终端/服务器
     │
 接入层：双归属、链路聚合、MSTP
     │
 汇聚层：VRRP、FW/LB双机、OSPF
     │
 核心层：双核心、三层互联、ECMP
```

### IPv6园区网演进图

```text
原IPv4园区网
      │
核心层、汇聚层升级双栈
      │
接入层保留二层设备
      │
部分纯IPv6用户通过隧道接入
      │
逐步演进到纯IPv6网络
```

## 3. 记忆口诀

### 口诀

**接入三角防环路，汇聚网关加路由；核心双机走三层，双栈隧道统层留；电网低延靠边缘，可靠切片保毫秒。**

### 解释

* **接入三角防环路**：接入—汇聚优先采用三角形组网，并通过MSTP避免二层环路。
* **汇聚网关加路由**：汇聚层通过VRRP保障网关冗余，通过OSPF保障三层路径冗余。
* **核心双机走三层**：核心层采用双核心、三层点到点互联和ECMP。
* **双栈隧道统层留**：IPv6升级采用双栈和隧道过渡；地址规划遵循统一、分层、预留原则。
* **电网低延靠边缘**：5G智能电网使用MEC减少传输距离和处理时延。
* **可靠切片保毫秒**：通过uRLLC切片满足低时延、高可靠的控制业务需求。
