````markdown
# 1.3 如何成为一名好的系统架构设计师

## 1. 本节核心思想

优秀架构师应兼具技术、系统、商业、领导与沟通能力，能够在业务需求、技术实现、成本风险和长期演进之间进行权衡。工程师成长为架构师，需要不断积累经验、拓宽视野、深入思考，并逐步形成系统化的架构设计方法论。

> **建议画图：是。** 本节角色较多、成长阶段较长，用图表示更容易理解和记忆。

    C --> D[初级架构师<br/>独立设计一般系统]
    D --> E[中级架构师<br/>设计复杂系统]
    E --> F[高级架构师<br/>创造新架构模式]
````

## 2. 考试高频考点

* **考点1：优秀架构师的六种角色特质。** 包括领导者、开发者、系统综合者、企业家、战略技术专家和沟通专家，既要懂技术，也要能决策、协调并推动方案落地。
* **考点2：工程师到架构师的成长阶段。** 依次为工程师、高级工程师、技术专家、初级架构师、中级架构师和高级架构师，核心变化是从实现功能发展到设计系统、解决复杂问题和创造新模式。
* **考点3：架构师的三项核心能力。** 判断力用于识别系统复杂度和脆弱点，执行力用于选择并落实合适方案，创新力用于创造新的复杂度解决方案。
* **考点4：架构工作的核心是权衡。** 架构师需要在性能与成本、敏捷与一致、当前需求与长期演进、技术先进性与团队能力之间取得平衡。
* **易混点：技术专家与初级架构师。** 技术专家主要依靠领域经验修改和优化架构；初级架构师则能够依据完整的架构设计方法论，说明方案选择的原因、约束和权衡过程。
* **易混点：工程师与架构师。** 工程师侧重具体设计、编码和测试，追求局部实现；架构师侧重业务抽象、系统建模、技术决策、团队协作及整体演进。
* **易考选择题：各成长阶段的典型特征。** 工程师是“指导下开发”，高级工程师是“独立开发”，技术专家是“领域专家”，初级架构师是“独立设计系统”，中级架构师是“设计复杂系统”，高级架构师是“创造新架构模式”。
* **易考选择题：架构师的成长原则。** 成长的内功来源于经验、视野和思考，应遵循“积累经验、拓宽视野、深度思考”，并持续进行学习、实践、总结和创新。

## 3. 记忆口诀

### 口诀一：六种角色

**领开系，企战沟。**

解释：

* **领**：领导者——统一技术愿景，建立信任并推动团队前进；
* **开**：开发者——理解代码、技术实现和现实约束，避免象牙塔式设计；
* **系**：系统综合者——综合考虑部署、性能、安全、测试、运维等系统质量；
* **企**：企业家——衡量技术选型的成本、收益和风险；
* **战**：战略技术专家——平衡长期战略与当前战术、敏捷度与一致性；
* **沟**：沟通专家——根据不同对象使用业务或技术语言，并做好决策记录。

### 口诀二：核心能力与成长原则

**判执创，经验宽，思考深。**

解释：

* **判**：判断复杂度和系统脆弱点；
* **执**：执行合适方案并推动落地；
* **创**：创新解决复杂问题的方法；
* **经验宽、思考深**：通过积累经验、拓宽视野和深度思考形成架构方法论。

### 口诀三：成长路线

**工高专，初中高；先会做，再会选，最终会创造。**

解释：

* 工程师、高级工程师、技术专家；
* 初级、中级、高级架构师；
* 能力由执行实现，发展到方案选择、系统权衡，最终创造新的架构模式。
