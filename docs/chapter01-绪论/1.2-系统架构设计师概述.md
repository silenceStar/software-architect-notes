# 1.2 系统架构设计师概述

## 1. 本节核心思想

系统架构设计师是项目的技术领导者，负责确定系统总体架构，处理非功能需求和关键技术问题，推动技术决策落地。其既要掌握业务、技术和设计知识，也要具备沟通、决策、协调和谈判能力。

## 2. 考试高频考点

* **考点1：架构设计师的定义与职责**
  架构设计师是负责系统架构的人、团队或组织，是项目的技术领导者，负责总体架构、关键构件、接口和非功能需求设计。

* **考点2：架构设计师的三大技术任务**
  从技术角度看，架构设计师主要承担抽象设计、非功能设计和关键技术设计。

* **考点3：架构设计师的主要任务**
  领导和协调技术活动，推动重大技术决策，确定系统架构，并形成需求、设计、实施和部署等架构文档。

* **考点4：架构设计师应具备的专业素质**
  主要包括业务知识、技术知识、设计能力、编程能力、沟通能力、决策能力、组织策略意识和谈判能力。

* **易混点：架构设计师与项目经理的区别**
  项目经理重点管理进度、成本、人员和资源；架构设计师重点负责技术方向、系统结构、质量属性和技术风险。

* **易混点：架构设计师不一定是一个人**
  架构设计师角色可以由个人或团队承担；团队承担时通常需要首席架构设计师统一协调和决策。

* **易混点：架构师不必精通所有技术细节**
  架构师应关注技术的关键因素、适用场景和整体影响，不要求熟悉平台中的每一个 API，但应至少精通一个技术领域。

* **易考选择题：架构师最重要的软技能**
  沟通能力是架构师最重要的软技能，用于理解利益相关者需求、传达架构思想并推动团队达成一致。

* **易考选择题：架构设计师的知识特点**
  架构师应具有较广的知识面和至少一个精通领域，即兼具技术广度与专业深度。

## 3. 记忆口诀

### 口诀

> **懂业务、定架构、保质量、解难题、善沟通、敢决策。**

### 解释

* **懂业务**：理解业务领域、用户需求和系统目标；
* **定架构**：划分系统结构，设计关键构件与接口；
* **保质量**：关注性能、安全、可靠性、可维护性等非功能需求；
* **解难题**：完成关键技术选型并控制技术风险；
* **善沟通**：协调利益相关者、开发人员和项目管理人员；
* **敢决策**：在信息不完全和存在冲突时及时作出技术取舍。

## 4. 考试一句话总结

> 系统架构设计师是项目的技术领导者，重点考查其三大技术任务、专业素质及其与项目经理的职责区别。
