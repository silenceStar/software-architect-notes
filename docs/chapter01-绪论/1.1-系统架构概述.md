# 1.1 系统架构概述

## 1. 本节核心思想

系统架构是系统的高层结构和整体蓝图，由组件、连接件、约束及设计演化原则构成，是连接需求分析与详细设计的桥梁。良好的架构能够保障系统的可靠性、安全性、可扩展性、可维护性等质量属性，并决定系统的健壮性和生命周期。

### 架构作用示意图

```text
用户需求
   ↓
需求分析
   ↓
系统架构设计
   ↓
详细设计与编码实现
   ↓
系统运行与持续演化
```

## 2. 考试高频考点

* **考点1：系统架构的定义**
  系统架构是系统的基本组织，包括组件、组件之间及其与环境之间的关系，以及指导系统设计和演化的原则。

* **考点2：架构设计的作用**
  架构设计位于需求分析之后、详细设计之前，主要解决复杂需求、非功能属性、系统集成、长期扩展及业务流程再造等问题。

* **考点3：模块化设计原则**
  模块划分应遵循高内聚、低耦合、粒度适中、接口简单、调用层次合理、信息隐藏和尽量复用等原则。

* **考点4：五种常见架构类型**
  分层架构强调职责分层；事件驱动架构通过事件异步解耦；微核架构采用核心加插件；微服务架构强调服务独立部署；云架构强调弹性扩展和分布式处理。

* **考点5：四种架构建模方法**
  结构模型描述组件和连接关系；框架模型描述系统整体框架；动态模型描述运行时行为与演化；过程模型描述系统构建步骤和开发过程。

* **考点6：“4+1”视图模型**
  包括逻辑视图、过程视图、开发视图、物理视图和场景视图，其中场景视图用于串联、说明和验证其他四个视图。

* **考点7：架构评估方法**
  SAAM用于分析架构对质量属性的支持；ATAM用于分析质量属性之间的权衡；CBAM在架构权衡的基础上进一步考虑成本与收益。

* **易混点：模块化、组件化与应用集成**
  模块化主要是逻辑上的功能切分；组件化强调可组装、可插拔和相对独立；应用集成则是把多个异构、独立的应用系统整合为统一系统。

* **易混点：系统架构与软件架构**
  系统架构范围更广，可包含软件、硬件、网络和运行环境；软件架构主要描述软件系统的高层组织结构。

* **易混点：分层架构与微服务架构**
  微服务解决整个系统如何拆分为独立服务，分层架构解决单个系统或服务内部如何按职责组织，两者可以同时使用。

* **易考选择题：架构风格与适用场景**
  分层架构适合职责清晰的管理系统；事件驱动适合异步消息和实时处理；微核适合插件式产品；微服务适合大型分布式系统；云架构适合高并发和弹性扩缩容。

* **易考选择题：常见架构风格分类**
  David Garlan和Mary Shaw将架构风格分为数据流、调用/返回、独立组件、虚拟机和仓库五类。

* **易考选择题：大型系统的架构选择**
  现代大型软件通常不会只采用一种架构风格，而是根据不同子系统和质量属性要求混合使用多种架构风格。

## 3. 记忆口诀

### 口诀一：架构基本定义

> **组、联、环、则**

解释：

* **组**：系统由哪些组件组成；
* **联**：组件之间如何连接和协作；
* **环**：系统与外部环境的关系；
* **则**：指导系统设计和演化的原则。

### 口诀二：五种常见架构

> **层事核微云**

解释：

* **层**：分层架构，按职责分层；
* **事**：事件驱动，通过事件解耦；
* **核**：微核架构，核心加插件；
* **微**：微服务架构，服务独立部署；
* **云**：云架构，弹性扩展和分布式处理。

### 口诀三：“4+1”视图

> **逻过开物，场景串联**

解释：

* **逻**：逻辑视图看系统功能；
* **过**：过程视图看运行、并发和通信；
* **开**：开发视图看代码和模块组织；
* **物**：物理视图看部署和网络节点；
* **场景**：通过用例验证并串联其他视图。

### 口诀四：模块划分原则

> **高内低耦，适度简复**

解释：

* 高内聚；
* 低耦合；
* 模块粒度适中；
* 接口简单；
* 尽量复用已有模块。

---

## 4. 考试一句话总结

系统架构是连接需求与实现的系统骨架，考试重点掌握其定义、质量属性、五种常见架构、四种建模方法及“4+1”视图模型。
