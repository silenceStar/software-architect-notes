# 10.5 软件架构演化评估方法

## 1. 本节核心思想（100字以内）

软件架构演化评估用于判断架构演化后质量属性是否达到预期。根据是否掌握演化过程，分为**演化过程已知评估**和**演化过程未知评估**。通过度量可维护性、可靠性等质量属性，计算架构质量属性距离，分析架构变化对质量的影响，监控架构健康演化。

---

## 2. 考试高频考点（尽量精简）

### （1）两种演化评估方式

- 演化过程已知：跟踪中间版本进行评估
- 演化过程未知：根据前后版本逆向推测演化过程

### （2）已知过程评估流程

```text
架构版本序列
→ 质量属性度量
→ 计算质量属性距离
→ 分析演化影响
```

### （3）可维护性度量（6项）

- 圈复杂度（CCN）
- 扇入（Fan-In）
- 扇出（Fan-Out）
- 响应集（RFC）
- 紧内聚（TCC）
- 松内聚（LCC）

> 可维护性度量结果是六维向量

### （4）可靠性度量

- 度量结果为单个实数值
- 可靠性距离：

```text
Dr(A,B)=(a-b)²/(a+b)
```

### （5）质量属性距离

- 可维护性：向量归一化欧氏距离
- 可靠性：归一化实数距离

### （6）重要结论

```text
D(0,n) ≠ D(0,1)+D(1,2)+...+D(n-1,n)
```

即：

- 架构质量属性距离不具有累加性
- 非相邻版本距离 ≠ 相邻版本距离之和

### （7）架构演化评估作用

- 修改影响分析
- 监控演化过程
- 分析关键演化节点
- 故障定位与风险控制

### （8）未知过程评估流程

```text
前后版本度量
→ 比较差异
→ 推测演化操作
→ 分析驱动因素
→ 验证是否达到演化目标
```

## 3. 记忆口诀（尽量精简）

### 两种评估

```text
已知跟踪，未知逆推
```

### 已知过程流程

```text
度量、算距、分析
```

### 可维护性六指标

```text
复杂两扇一响应，
紧松内聚共六项
```

（复杂=CCN，两扇=Fan-In/Fan-Out，响应=RFC，紧松=TCC/LCC）

### 评估作用

```text
分析、监控、定位
```

### 必背结论

```text
质量距离不可加
```

即：

```text
D(0,n) ≠ ΣD(i,i+1)
```
