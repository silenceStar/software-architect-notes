# 10.4 软件架构演化原则

## 1. 本节核心思想（100字以内）

软件架构演化应遵循成本、进度、风险可控的原则，在保证系统主体稳定的前提下，通过低耦合、低复杂度、高复用的方式持续优化架构，使系统能够适应新技术、新环境和新需求，并不断提升软件质量，实现可持续演化。

---

## 2. 考试高频考点（尽量精简）

### 六大类十八原则

#### （1）控制类
- 演化成本控制（ECC）
- 进度可控
- 风险可控
- 目标一致

#### （2）平稳演化类
- 主体维持（AIG）
- 平滑演化（IWR）
- 总体结构优化

#### （3）影响控制类
- 模块独立演化（Local Change）
- 影响可控（Impact Limitation）

#### （4）可维护性类
- 复杂性可控
- 有利于重构
- 有利于重用（高内聚、低耦合）

#### （5）规范遵从类
- 设计原则遵从性
- 适应新技术（TI）
- 标准依从性

#### （6）适应能力类
- 环境适应性
- 质量向好（QI）
- 适应新需求

### 高频度量指标
- ECC：CoE << CoRD
- 进度：ttask = |Ttask - T'task|
- AIG = 变更量 / 主体规模
- IWR = 变更总量 / 项目规模
- RCP = 冲突设计原则数 / 设计原则总数
- TI = 1 - DDT
- EQI > SQ
- RNR = |ANR / NR|

## 3. 记忆口诀（尽量精简）

### 分类口诀

**四控三稳两影响，三易三规三适应。**

### 对应关系

- 四控：成本、进度、风险、目标
- 三稳：主体维持、平滑演化、结构优化
- 两影响：局部修改、影响可控
- 三易：易维护（复杂度）、易重构、易复用
- 三规：设计原则、技术独立、标准遵从
- 三适应：环境适应、质量向好、需求适应

### 考场速记

**控成本、控进度、控风险；**
**保稳定、降影响、减复杂；**
**重复用、守规范；**
**适环境、提质量、应需求。**
