# 10.2 面向对象软件架构演化过程

## 1. 本节核心思想（100字以内）

面向对象软件架构演化主要体现在顺序图元素的变化，包括对象、消息、复合片段和约束四类演化。其中消息演化是核心，对象演化通常伴随消息演化发生，复合片段演化影响控制流程，约束演化影响系统正确性，需要验证演化后架构是否满足约束要求。

---

## 2. 考试高频考点（尽量精简）

### （1）对象演化
- AO（Add Object）：增加对象
- DO（Delete Object）：删除对象
- 对象演化通常伴随消息演化，否则无意义

### （2）消息演化（重点）
- AM（Add Message）：增加消息
- DM（Delete Message）：删除消息
- SMO（Swap Message Order）：交换消息顺序
- OM（Overturn Message）：反转消息方向
- CMM（Change Message Module）：改变消息发送方或接收方

### （3）复合片段演化
- AF（Add Fragment）：增加复合片段
- DF（Delete Fragment）：删除复合片段
- FTC（Fragment Type Change）：改变复合片段类型
- FCC（Fragment Condition Change）：改变复合片段条件

### （4）约束演化
- AC（Add Constraint）：增加约束
- DC（Delete Constraint）：删除约束
- 修改约束 = DC + AC

### （5）重点关系
- 消息演化是顺序图演化核心
- 对象、复合片段、约束演化均与消息演化密切相关
- 演化后需验证架构正确性和时态属性
