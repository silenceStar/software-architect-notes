# 18.6 数据库系统的安全设计

## 1. 本节核心思想（100字以内）

数据库安全设计旨在保证数据的**机密性、完整性和可用性（CIA）**。安全设计主要包括用户管理、访问控制、数据加密、审计和攻击检测；完整性设计则通过各种完整性约束保证数据正确性和一致性，是数据库设计的重要组成部分。

## 2. 考试高频考点

- **考点1：数据库安全评估标准**
  - TCSEC（橘皮书）是计算机安全评估标准；TDI是TCSEC在数据库管理系统上的扩展。安全等级分为 **D、C1、C2、B1、B2、B3、A1**，其中**B1及以上才属于真正意义上的安全产品**。

- **考点2：数据库完整性设计**
  - 数据库完整性是保证数据**正确性和相容性**，主要依靠各种完整性约束（主键、外键、非空、唯一、Check、Trigger）来实现。

- **易混点：数据库安全 vs 数据库完整性**
  - **数据库安全**：防止数据泄露、非法访问和攻击（保密性）。
  - **数据库完整性**：保证数据合法、正确、一致（正确性）。

- **易考选择题**
  - ① TCSEC中哪个等级以上属于真正安全产品？（**B1及以上**）
  - ② TDI是什么？（**TCSEC在数据库领域的扩展解释**）
  - ③ Oracle支持哪些完整性约束？（Not Null、Unique、Primary Key、Foreign Key、Check、Trigger）
  - ④ 静态约束与动态约束分别由谁实现？（**静态约束优先DBMS；动态约束一般应用程序实现**）
  - ⑤ 为什么慎用Trigger？（**性能开销大、多级触发难控制**）

## 3. 记忆口诀

### 口诀

**安全五件套，完整六约束；B一才安全，静库动程序。**

### 解释

- **安全五件套**：用户管理、存取控制、数据加密、审计跟踪、攻击检测。
- **完整六约束**：Not Null、Unique、Primary Key、Foreign Key、Check、Trigger。
- **B一才安全**：TCSEC中只有 **B1及以上** 才是真正意义上的安全产品。
- **静库动程序**：静态完整性约束尽量由数据库实现，动态约束一般由应用程序实现。

## 4. 考试一句话总结（精简）

**重点考TCSEC/TDI安全等级、数据库完整性约束（主键、外键、非空等）及完整性设计原则，尤其牢记"B1及以上是真正安全产品，静态约束交数据库、动态约束交应用程序"。**
