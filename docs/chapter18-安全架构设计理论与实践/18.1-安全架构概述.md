# 18.1 安全架构概述

## 1. 本节核心思想

安全架构以风险和安全策略为基础，从技术、管理、人员和工程过程等方面进行整体防护，通过产品安全、安全技术体系和审计三道防线，保障信息系统的机密性、完整性、可用性、可控性和不可抵赖性。

## 2. 考试高频考点

* **考点1：信息系统安全威胁的四大来源**
  安全风险主要来自人为蓄意破坏、灾害性攻击、系统故障和人员无意识行为，其中人为攻击又可分为主动攻击和被动攻击。

* **考点2：安全威胁涉及的六个层面**
  威胁可能出现在物理环境、通信链路、网络系统、操作系统、应用系统和管理系统，说明安全设计必须从整体系统角度考虑，不能只关注网络层。

* **考点3：安全架构的三道防线**
  产品安全架构强调产品自身安全，安全技术体系架构提供统一安全基础设施，审计架构负责独立发现风险、追踪行为和落实责任。

* **考点4：安全架构的核心属性**
  机密性防止信息泄露，完整性防止数据被未授权修改，可用性保证合法用户正常使用系统；此外还包括可控性和不可抵赖性。

* **考点5：主要安全技术措施**
  安全架构常采用身份鉴别、访问控制、内容安全、冗余恢复、审计响应、恶意代码防范和密码技术等手段。

* **易混点1：主动攻击与被动攻击**
  被动攻击只获取信息，通常不改变系统状态，如窃听和业务流分析；主动攻击会篡改、伪造或破坏系统，如假冒、重放、数据篡改和拒绝服务。

* **易混点2：木马、病毒与陷阱门**
  木马通过伪装诱导用户执行，病毒具有感染和传播能力，陷阱门则是系统中预留的秘密入口或后门。

* **易混点3：身份鉴别与访问控制**
  身份鉴别解决“你是谁”，访问控制解决“你可以做什么”。

* **易考选择题1：典型威胁对应的安全属性**
  信息泄露主要破坏机密性，数据篡改主要破坏完整性，拒绝服务主要破坏可用性，抵赖主要破坏不可抵赖性。

* **易考选择题2：典型安全标准**
  TCSEC称为“橘皮书”；CC是信息技术安全评价通用准则，并进一步形成ISO/IEC 15408；我国GB 17859用于安全保护等级划分，GB/T 18336用于信息技术安全评估。

* **易考选择题3：国内标准缩写**
  GB表示国家标准，GB/T表示推荐性国家标准，GA表示公共安全行业标准，GJB表示国家军用标准。

## 3. 记忆口诀

**“四类威胁六层防，产品技术审计查；机密完整保可用，鉴权加密防抵赖。”**

解释：

* “四类威胁”：人为破坏、灾害、系统故障、无意识行为；
* “六层防”：物理、链路、网络、操作系统、应用、管理；
* “产品技术审计”：产品安全、安全技术体系、审计三道防线；
* “机密完整保可用”：CIA三大核心安全属性；
* “鉴权加密防抵赖”：身份鉴别、访问控制、密码技术和审计机制。

补充口诀：

**“被动偷看，主动动手；木马伪装，病毒传播，陷阱后门。”**

用于区分主动与被动攻击，以及木马、病毒、陷阱门三类常见威胁。

## 4. 考试一句话总结

本节最可能考安全威胁分类、主动与被动攻击区别、CIA三要素、安全架构三道防线，以及TCSEC、CC、ISO/IEC 15408、GB 17859和GB/T 18336等典型标准。
