# 18.8 安全架构设计案例分析

## 1. 本节核心思想（100字以内）

本节通过RADIUS和混合云工业系统两个案例，说明安全架构设计必须结合具体业务，在实现认证授权、数据保护的同时，兼顾高并发、高可用、可扩展和业务连续性，形成分层、解耦、纵深防御的安全体系。

## 2. 考试高频考点

* **考点1：RADIUS的AAA机制**
  AAA包括认证、授权和审计（记账）：认证判断用户能否访问，授权决定用户能使用哪些服务，审计记录用户资源使用情况。

* **考点2：RADIUS三层软件架构**
  协议逻辑层负责通信协议和报文转发，业务逻辑层负责认证、授权、计费，是架构核心；数据逻辑层通过数据库代理池统一访问数据库。

* **考点3：RADIUS高性能与高可用设计**
  通过多进程、多线程、共享内存、数据库代理池和负载均衡，提高并发处理能力、响应速度、资源利用率，并减少单点故障。

* **考点4：混合云工业系统的安全重点**
  工业安全架构重点保护设备、网络、控制、应用和数据五个方面，形成覆盖物理设备、生产控制、业务应用和数据生命周期的纵深防御。

* **易混点**
  认证解决“用户是谁、能否进入”，授权解决“进入后能做什么”，审计解决“做了什么、使用了多少资源”；负载均衡既能提高性能，也能提高可用性。

* **易考选择题**
  RADIUS架构的核心是业务逻辑层；数据库代理池用于降低数据库压力和系统对数据库的依赖；混合云通常将核心敏感业务放在私有云，将公共、弹性业务放在公有云。

### 辅助理解图

```text
RADIUS案例：
用户 → BAS → 协议逻辑层 → 业务逻辑层 → 数据逻辑层 → 数据库
                       认证 / 授权 / 计费

混合云工业案例：
设备层 → 控制层 → 设计管理层 → 云应用层
  │        │           │          │
设备安全  控制安全    应用安全    数据安全
       └──────── 网络安全 ────────┘
```

## 3. 记忆口诀

### 口诀

**“认授审，协业务数；设网控用数，公弹私核心。”**

### 解释

* **认授审**：认证、授权、审计，即AAA。
* **协业务数**：RADIUS分为协议逻辑层、业务逻辑层、数据逻辑层。
* **设网控用数**：工业系统五大安全重点是设备、网络、控制、应用、数据安全。
* **公弹私核心**：公共、弹性业务适合公有云，核心、敏感业务适合私有云。

## 4. 考试一句话总结（精简）

本节最可能考AAA的区别、RADIUS三层架构及其高性能设计，以及混合云工业系统“设备、网络、控制、应用、数据”五大安全重点。
