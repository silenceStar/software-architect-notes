# 18.5 网络安全体系架构设计

## 1. 本节核心思想

OSI安全体系架构以**鉴别、访问控制、数据机密性、数据完整性和抗抵赖**五类安全服务为核心，通过八类安全机制，并采用分层、多点、纵深防御方式，将安全能力部署到网络、边界、计算环境及支撑基础设施中。

```text
网络安全体系架构
├─ 五类安全服务：鉴别、访问控制、机密性、完整性、抗抵赖
├─ 八类安全机制：加密、签名、访问控制、完整性、鉴别交换、
│                业务流填充、路由控制、公证
└─ 深度防御体系
   ├─ 多点防御：网络、边界、计算环境
   ├─ 分层防御：多道独立安全屏障
   └─ 支撑设施：PKI、检测与响应基础设施
```

## 2. 考试高频考点

* **考点1：OSI五类安全服务与八类安全机制**
  五类安全服务是安全目标，八类安全机制是实现手段；最适合配置安全服务的是物理层、网络层、运输层和应用层。

* **考点2：分层多点安全技术体系架构**
  包括网络与基础设施、边界、计算环境三个防御区域，以及PKI、检测与响应基础设施等公共支撑能力。

* **考点3：认证与访问控制框架**
  认证解决“你是谁”，可依据口令、令牌、生物特征、可信第三方和环境；访问控制中ADF负责判决，AEF负责执行。

* **考点4：机密性、完整性与抗抵赖**
  机密性防泄露，完整性防止或检测未授权修改、删除和重放，抗抵赖通过证据生成、保存、验证和纠纷解决防止事后否认。

* **易混点**
  安全服务是“要实现什么”，安全机制是“如何实现”；鉴别确认身份，访问控制判断权限；ADF负责判决，AEF负责实施；数字签名主要保证身份真实性、完整性和抗抵赖，本身不保证机密性。

* **易考选择题**
  ADF的输入包括发起者ADI、目标ADI、访问请求ADI、访问控制策略和上下文信息；ADI是ACI中实际用于本次访问控制判决的部分。

## 3. 记忆口诀

**五服八机，四层适配；三点两撑，鉴判密整赖。**

解释：

* **五服**：鉴别、访问控制、机密性、完整性、抗抵赖。
* **八机**：加密、数字签名、访问控制、数据完整性、鉴别交换、业务流填充、路由控制、公证。
* **四层适配**：物理层、网络层、运输层、应用层最适合配置安全服务。
* **三点**：网络与基础设施、边界、计算环境。
* **两撑**：PKI公钥基础设施、检测和响应基础设施。
* **鉴判密整赖**：认证身份、判定权限、防止泄露、防止篡改、防止抵赖。

## 4. 考试一句话总结

本节最可能考查OSI的**五类安全服务、八类安全机制、分层多点防御体系，以及ADF与AEF等关键概念的区别**。
