## 1. 本节核心思想（100字以内）

安全模型是将抽象的安全策略具体化，规定主体对客体的访问规则，实现系统的机密性、完整性和访问控制。考试重点掌握五种典型安全模型的保护目标、核心规则及适用场景，尤其是 **Bell-LaPadula** 与 **Biba** 的读写规则区别。

### 五种模型对比图

| 模型 | 保护目标 | 核心思想 | 典型场景 |
|------|---------|---------|---------|
| 状态机 | 系统安全状态 | 始终保持安全状态转换 | 安全系统基础 |
| BLP | **机密性** | 不上读、不下写 | 军事、涉密系统 |
| Biba | **完整性** | 不下读、不上写 | 高可信数据 |
| Clark-Wilson | 完整性 | 程序控制+职责分离+审计 | 银行、财务 |
| Chinese Wall | 利益冲突 | 同一竞争组只能访问一家 | 投资银行、咨询公司 |

## 2. 考试高频考点

- **考点1：Bell-LaPadula（BLP）模型**
  - 保护**机密性**，遵循 **No Read Up（不上读）**、**No Write Down（不下写）**，防止高密级信息泄露。

- **考点2：Biba模型**
  - 保护**完整性**，遵循 **No Read Down（不下读）**、**No Write Up（不上写）**，防止低可信数据污染高可信数据。

- **易混点**
  - **BLP保护机密性，Biba保护完整性；两者读写规则正好相反。**
  - Clark-Wilson也是完整性模型，但强调**事务处理、职责分离、程序控制、审计**，不是按安全级别控制。
  - Chinese Wall解决的是**利益冲突**，权限会随着历史访问动态变化。

- **易考选择题**
  - 哪个模型用于军事保密？（BLP）
  - 哪个模型用于银行事务处理？（Clark-Wilson）
  - 哪个模型防止利益冲突？（Chinese Wall）
  - 哪个模型强调"安全状态始终保持安全"？（状态机模型）
  - BLP与Biba的"No Read/Write"规则对应关系。（高频）

## 3. 记忆口诀

### 口诀

> **状态保安全，BLP保机密；  
> Biba保完整，银行CWM；  
> 中国墙防竞争，读写方向要分清。**

### 解释

- **状态保安全**：状态机模型始终保持安全状态。
- **BLP保机密**：不上读（No Read Up）、不下写（No Write Down）。
- **Biba保完整**：不下读（No Read Down）、不上写（No Write Up）。
- **银行CWM**：Clark-Wilson强调事务、职责分离、程序控制和审计。
- **中国墙防竞争**：同一利益冲突组只能访问一家企业。
